# 要件定義書: Web Serial Plotter

## 1. プロジェクト概要

WEBブラウザ上で動作するシリアル通信データのリアルタイムプロッターを開発する。  
シリアルポートから受信したASCII形式の時系列データを解析し、グラフ上にリアルタイムで描画するスタティックWebアプリケーションである。

## 2. システム構成

### 2.1 動作環境

| 項目 | 内容 |
|------|------|
| 動作環境 | Webブラウザ上（クライアントサイドのみ） |
| 対応ブラウザ | Web Serial API対応ブラウザ（Google Chrome, Microsoft Edge） |
| 開発言語 | TypeScript |
| サーバ要件 | なし（HTML / CSS / TypeScriptをビルドしたスタティックサイト） |
| 通信方式 | Web Serial API を使用 |

### 2.2 技術的前提条件

- バックエンドサーバは用意しない
- すべての処理はクライアントサイドで完結する
- 開発言語として TypeScript を使用し、ビルド成果物（HTML / CSS / JavaScript）のみで構成されるスタティックサイトとする
- Web Serial API を利用してシリアルポートへアクセスする
- Web Serial API は HTTPS または localhost 環境でのみ動作する

## 3. 機能要件

### 3.1 シリアル通信機能

#### 3.1.1 接続管理

| 項目 | 内容 |
|------|------|
| ポート選択 | ブラウザのシリアルポート選択ダイアログを通じてポートを選択できる |
| ボーレート設定 | 任意のボーレートを指定できる（一般的な値をプリセットとして用意する） |
| データビット設定 | 7 / 8 ビットを選択できる |
| ストップビット設定 | 1 / 2 ビットを選択できる |
| パリティ設定 | none / even / odd を選択できる |
| フロー制御設定 | none / hardware を選択できる |
| 接続状態表示 | 接続中 / 切断中の状態を画面上に表示する |
| 切断操作 | 接続中のポートを切断できる |

#### 3.1.2 データ送信機能

| 項目 | 内容 |
|------|------|
| 入力形式 | テキスト形式およびバイナリ形式（16進数表記）の切り替えが可能 |
| 改行コード付与 | 末尾に付与する改行コードを選択できる（なし / `\r` / `\n` / `\r\n`） |
| 送信履歴表示 | 送信した内容を送信時刻とともに一覧表示し参照できる |

#### 3.1.3 データ受信機能

| 項目 | 内容 |
|------|------|
| 受信データ形式 | ASCII文字列: `{key1}:{value1},{key2}:{value2},...\n` |
| 受信処理 | リアルタイムに受信し、受信時刻（ブラウザ側のタイムスタンプ）を記録する |
| 区切り文字 | 行区切り: `\n`（LF）、キー・バリュー区切り: `:`、データ項目区切り: `,` |

**受信データの具体例:**
```
temperature:25.6,humidity:43
```
上記の場合、`temperature` = `25.6`、`humidity` = `43` として解析される。

### 3.2 リアルタイムグラフ描画機能

#### 3.2.1 グラフの基本仕様

| 項目 | 内容 |
|------|------|
| 横軸 | 受信時刻（時間軸） |
| 縦軸 | 測定データの値 |
| 描画方式 | 最新値が右端にプロットされ、常に左方向へスクロールする（リアルタイムストリーミング表示） |
| 更新タイミング | データ受信の有無にかかわらず、指定されたFPSで定期的に更新する |

#### 3.2.2 測定の開始と停止

| 項目 | 内容 |
|------|------|
| 測定開始 | 「測定開始」ボタンを押下するとグラフ描画を開始する |
| 測定停止 | 「測定停止」ボタンを押下するとグラフ描画を停止する |
| データ受信との独立性 | シリアルポートが接続されていればデータ受信は常時行われ、グラフ描画の開始/停止はグラフの更新のみを制御する |

#### 3.2.3 FPS設定

| 項目 | 内容 |
|------|------|
| 設定方法 | スライダーで指定する |
| 最小値 | 1 FPS |
| 最大値 | 60 FPS |
| 現在値表示 | スライダー付近に現在の設定値を表示する |

#### 3.2.4 横軸（時間幅）設定

| 項目 | 内容 |
|------|------|
| 設定方法 | スライダーで調整する |
| 最小値 | 1 秒間 |
| 最大値 | 60 秒間 |
| 現在値表示 | スライダー付近に現在の設定値を表示する |
| 動作 | 設定した秒数分のデータがグラフ横軸に表示される |

#### 3.2.5 縦軸設定

| 項目 | 内容 |
|------|------|
| 軸の種類 | 第1軸（左側）と第2軸（右側）の2軸構成 |
| 表示範囲モード | 自動（受信データに基づき自動算出）または手動（ユーザーが最小値・最大値を入力） |
| 軸の独立性 | 第1軸と第2軸はそれぞれ独立して表示範囲を設定できる |

### 3.3 データ一覧表示機能

| 項目 | 内容 |
|------|------|
| 表示内容 | 受信データのキー名と最新値をテーブル形式で一覧表示する |
| 軸割り当て | データの種類（キー）ごとに第1軸（左）/ 第2軸（右）のどちらに紐づけるかを選択できる |
| 最新値表示 | 各キーの最新値をテキストとして表示する |
| 更新タイミング | データ受信時にリアルタイムで更新される |

## 4. 非機能要件

### 4.1 UI / デザイン

| 項目 | 内容 |
|------|------|
| デザイン方針 | モダンなデザイン（ダークテーマベース、視認性の高い配色） |
| レスポンシブ対応 | デスクトップブラウザでの利用を主対象とする |
| レイアウト | グラフ表示領域を中心に、設定パネルとデータ一覧を配置する |

### 4.2 パフォーマンス

| 項目 | 内容 |
|------|------|
| グラフ描画性能 | 60FPS設定時にもスムーズに描画できること |
| データ処理 | 受信データの解析・蓄積がグラフ描画をブロックしないこと |

### 4.3 ユーザビリティ

| 項目 | 内容 |
|------|------|
| 操作性 | 直感的な操作フローでシリアル接続からグラフ描画まで到達できること |
| エラー通知 | 接続失敗・データ解析エラー等をユーザーに分かりやすく通知すること |
| Web Serial API 非対応時 | 非対応ブラウザで開いた場合、その旨を表示すること |

## 5. 画面構成（概要）

```
+------------------------------------------------------------------+
|  ヘッダー（タイトル）                                              |
+------------------------------------------------------------------+
|  接続設定パネル                                                    |
|  [ポート選択] [ボーレート] [接続/切断ボタン] [接続状態]              |
+------------------------------------------------------------------+
|                                                                    |
|  グラフ表示領域                                                    |
|  （リアルタイムプロット / 第1軸(左) + 第2軸(右)）                    |
|                                                                    |
+------------------------------------------------------------------+
|  グラフ設定: [測定開始/停止] [FPSスライダー] [時間幅スライダー]      |
|  縦軸設定: [第1軸: 自動/手動] [第2軸: 自動/手動]                    |
+------------------------------------------------------------------+
|  データ一覧テーブル           |  送受信パネル                      |
|  キー名 | 最新値 | 軸選択    |  [送信入力欄] [送信履歴]           |
+------------------------------------------------------------------+
```

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| Web Serial API | ブラウザからシリアルポートへアクセスするためのWeb標準API |
| FPS | Frames Per Second。グラフの1秒あたりの描画回数 |
| 第1軸 | グラフ左側の縦軸 |
| 第2軸 | グラフ右側の縦軸 |
| キー | 受信データにおけるデータ項目名（例: `temperature`, `humidity`） |
| バリュー | 受信データにおけるデータ項目の値（例: `25.6`, `43`） |
